Imagine a nationwide emergency radio network. You’re given a list of two-way radio links, where each link connects a pair of base stations that can communicate directly. Your goal is to compute the network’s effective relay diameter based on shortest relay chains.

The set of stations consists of all unique names that appear in the link list. A coverage cluster is a largest possible group of stations in which every pair of stations is connected by some sequence of links. The relay distance between two stations is the minimum number of link hops on any chain connecting them within the same cluster. For any connected cluster C, its relay diameter is the maximum such shortest-hop length over all station pairs if C has at least two stations, and 0 if C has exactly one station. The effective relay diameter of the overall network is defined as follows: if the entire network is connected, it is the network’s relay diameter; otherwise, select any coverage cluster with the most stations (ties broken arbitrarily) and take that cluster’s relay diameter. If the network contains no stations, no value is reported.

Return the network’s effective relay diameter as a nonnegative integer, or return no value if there are no stations.
