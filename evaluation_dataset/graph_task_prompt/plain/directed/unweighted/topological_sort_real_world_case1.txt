A software build system represents file compilation dependencies as a directed graph ( G = (V, E) ), where each node corresponds to a source file and each edge ((u, v)) indicates that file (u) must be compiled before file (v). To produce a valid build sequence, your task is to compute a topological ordering of this dependency graph. This ordering ensures that all prerequisite files are compiled before the files that depend on them. If no such ordering exists, the dependency graph contains a circular dependency that must be resolved before compilation can proceed.
