Bron–Kerbosch algorithm (backtracking); The idea is to grow a candidate clique by repeatedly adding a node that connects to all nodes already in the clique, backtracking when no further expansion is possible, and keeping the largest clique found.

Example 1: Find the maximum clique
Graph: nodes {1, 2, 3, 4, 5}, edges: (1,2), (1,3), (2,3), (2,4), (3,4), (4,5)
Step 1: Start with an empty clique R = {}, candidate set P = {1,2,3,4,5}, excluded set X = {}.
Step 2: Pick a node from P, say 2, and try to include it: R = {2}. Now restrict candidates to neighbors of 2: P = {1,3,4} (since 2 connects to 1,3,4).
Step 3: From P, pick 3 and include it: R = {2,3}. Restrict candidates to nodes that are neighbors of both 2 and 3: P = {1,4} (1 connects to 2 and 3; 4 connects to 2 and 3).
Step 4: From P, pick 1 and include it: R = {1,2,3}. Now restrict candidates to neighbors of all of {1,2,3}. There are none left (P becomes empty), so {1,2,3} is a maximal clique of size 3. Record best = size 3 so far.
Step 5: Backtrack: remove 1 from the current choice and try the next candidate 4 instead. Include 4: R = {2,3,4}. Again, no further node connects to all of {2,3,4} (P becomes empty), so {2,3,4} is another maximal clique of size 3. Best remains 3.
Step 6: Continue backtracking to explore other starts (like starting from 4 leads to {4,5} of size 2), but no clique larger than size 3 appears.
Step 7: Conclude the maximum clique size is 3, and one maximum clique is {1,2,3} (another is {2,3,4}).
